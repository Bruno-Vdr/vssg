vssg's commands documentation:


Command: vssg init blog_name

The init command initializes a new blog:
    -Creates a directory with the given blog_name.
    -Creates a .blog config file inside this directory.
    -Generates a default style.css to be used by local HTML.
    -Generates a default topics_list.tmpl topic list template file.

________________________________________________________________________________

Command: vssg add topic

Warning: This command must be launched from within blog directory.

The add command creates a new topic inside the blog:
    -Create a directory based on hashed(topic).
    -Update/create .topic file in hashed directory.
    -Update .blog in blog's directory.
    -Create style.css in topic's directory.
    -Create pushs_list.tmpl in topic's directory.
    -Create push.tmpl in topic's directory.
    -Create push_style.tmpl in topic's directory.
    -Generate base.htm HTML file, with links to differents topics.
    -Generate in new topic directory a index.htm (Empty) html page with pushes list.

________________________________________________________________________________

Command: vssg push push_text_file

Warning: This command must be launched from within topic directory.

The push command creates a new push/entry in the current topic directory:
    -Create a directory named push_xx
    -Create a pictures sub-directory named push_xx/./pictures
    -Update/Create the .topic with new push.
    -Create style.css in new directory push_xx for local customization
    -Generation of HTML push page index.htm based on template {push.tmpl}
    -Pictures copy to push_xx/./pictures based on push_text_file

________________________________________________________________________________

Command: vssg env

The env command displays the environment variables used by vssg:

    VSSG_PUSH_DIR : Default directory to get push files from.
    VSSG_IMG_PUSH_DIR : Path to grab pushed images from.
    VSSG_BLOG_REMOTE_URL : Remote blog's URL (used by sync command).
    VSSG_BLOG_ROOT : Local blog' location (used by sync command).
    VSSG_RSYNC_OPT : Permanent customizable rsync option e.g. "-e 'ssh -p 2223'".

________________________________________________________________________________

Command: vssg show [-a]

The show command displays information depending current working directory:
    In the blog root directory (containing .blog it displays:
    -topic list.

    With the -a option, topic's push are also displayed.

    In a topic directory (containing .topic it displays:
    -push list.

Command: vssg modify id push_text_file

Warning: This command must be launched from within topic directory.

The modify command modifies the push, identified by id with the given push file.
To get the push's id, just do "vssg show"
Modify is used to modify text, date, title or images of an already existing push.
    -The .topic file is updated accordingly to push_text_file
    -The push HTML code is rebuilt.
    -The HTML topic page with link to pushes is also rebuilt.

________________________________________________________________________________

Command: vssg remove id [-f]

Warning: This command must be launched from within topic directory.
To get the push's id, just do "vssg show"

The remove command deletes a push from a topic:
    -Removes push description from .topic
    -Regenerates index.htm with links to push.
    -Print out command to delete remaining directories.

Note: the remove commands only remove push from .topic. Directory push_ is
      not delete neither its contained files, subdirectory and images, its suffixed with __DELETED
      UNLESS -f is passed on the command line.

Command: vssg update

    The update command force generation of links page (topic list or push list) depending
    on the current directory. This command is usefull when modifying local template_list
    model.

    In the blog root directory (containing .blog) it rebuilds topic list.
    In a topic directory (containing .topic) it rebuilds push list.

Command: vssg rename title new_title

Warning: This command must be launched from within blog directory.

The rename command changes the title and directory of an already existing TOPIC:
    -Patch the topic title/comment into .blog file, containing topic(s) list.
    -Rename original topic directory with hash("new_title").
    -Update topic's .topic with new name and directory, keeping posts untouched.
    -Rebuild HTML topic list page "base.htm".

Command: vssg drop topic_title [-f]

Warning: This command must be launched from within blog directory.

The drop command deletes a complete topic with all of its pushes, if any. By default the commands appends the
__DELETED suffix to the directory. By adding the -f the
directory will be definitely removed.

    -It also updates .blog accordingly.
    -Rebuilt the HTML links to topic page, "base.htm".

To get a list of topics, run "vssg show" from blog's root directory."

Command: vssg sync [-bend] [-dry]

The sync command performs a synchronization between local and remote directories:
Source is defined with the env var VSSG_BLOG_ROOT  (set to: /home/vedder/Bruno-Archive/Development/V/vssg/Blog)
Destination is defined with the env var VSSG_BLOG_REMOTE_URL  (set to: daza1548@golem.o2switch.net:cam2025.vedder.fr)

The -dry option prevents command execution, and only prints the rsync command(s).
The -bend option also synchronizes the redirection file index.htm in blog root's directory, updated by bend command.

Permanent or additional option(s) can be set using environment variable VSSG_RSYNC_OPT e.g.  "-e 'ssh -p 2223'" to specify
a different SSH port with rsync under sync command.

Command: vssg bend [URL] [-f] [-u]

Launched from a topic directory, the bend command creates a blog redirection to its last push.

If an optional URL is provided, it will be used instead of last topic. This optional URL
must be relative to your location when the the command is run, unless -f option is used .

Adding the -f option, discards destination presence check. Use this to redirect on exterior URL.
Adding the -u option will sync/send the redirection file to the remote blog.

This command si usually used to bend blog's entry to the last push. It can be used to redirect to any other URL,
in case of unavailability for example.

    e.g. vssg bend -u will redirect to the last push in topic and sync the redirection.
    e.g. vssg bend -f https://duckduckgo.com/ will redirect to duckduckgo site.

Command: vssg pull destination_dir  -f

Warning: This command must be launched from outside blog directory.

The -f option, allows to continue if destination directory already exists. It's
useful to resume interrupted earlier pull command.

The pull command downloads the full remote blog to local the directory. This is useful
if the local blog is lost. This command is potentially dangerous, and must be run from
outside blog directory.

Command: vssg chain string

Warning: This command must be launched from within topic directory.

This command will open all pushe from the current topics, and insert (if any) a link to previous and
next push. Its done by patching custom HTML tag in template "<vssg-lnk-next>" and "<vssg-lnk-prev>".
Order is given by order of push in .topic file.

________________________________________________________________________________

Command: vssg backup directory

Warning: This command must be launched from outside blog's directory.

The backup command generates a compressed (.zip) archive if the blog in directory.
The archive can be restored later. Useful in case of dangerous command usage.
Note: No check is done to verify presence of a blog. The directory may contain anything.
Classic images format file are just stored .jpg .png as they are already compressed, to
save time at compression stage.

________________________________________________________________________________

Command: vssg obfuscate string

The obfuscate command applies obfuscation function (hash) over the given string. It allows
to retrieve topics mangled named.
e.g. vssg obfuscate vssg will return "dec5fe0145ac7440"

________________________________________________________________________________

Command: vssg base64 filename

The base64 command converts given image file to base64 string on std output. It allows images to be embedded in HTML
page, without need of additional file. HTML <img> tag is also creates, with complete mime type, and extension.

________________________________________________________________________________

Command: vssg deploy

Warning: This command must be launched from blog or topic directory.

The deploy command emits local  (embedded) templates files in the current directory accordingly to current blog or topic position.
Note: All files are deployed by the command.
